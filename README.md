## รายละเอียดเกม
เกมหนีรอดในเขาวงกตที่ผู้เล่นต้องวิ่งเก็บเหรียญให้ครบ 3 เหรียญเพื่อผ่านด่าน โดยความยากจะเพิ่มขึ้นตามเลเวล ทั้งความเร็วของปีศาจและจำนวนที่เพิ่มขึ้น (สูงสุด 4 ตัว) ตัวเกมมีลูกเล่นเพิ่มเติมเพื่อเพิ่มความท้าทาย เช่น ประตูมิติวาร์ปข้ามฝั่ง และไอเทมลูกแก้ววิญญาณที่ช่วยให้เดินทะลุกำแพงได้ชั่วคราว

## วิธีการเล่นและการควบคุม
- ใช้ปุ่ม **ลูกศร (Arrow Keys)** หรือ **W, A, S, D** เพื่อเคลื่อนที่ตัวละคร
- **เหรียญ (สีทอง):** เก็บให้ครบ 3 เหรียญเพื่อผ่านด่าน
- **ประตูมิติ (เรืองแสงสีฟ้า):** เดินเข้ามุมซ้ายบนหรือขวาล่าง เพื่อวาร์ปข้ามไปอีกฝั่งของแผนที่
- **ลูกแก้ววิญญาณ (สีฟ้าเขียว):** จะปรากฏตั้งแต่เลเวล 3 ขึ้นไป เมื่อเก็บแล้วจะเข้าสู่สถานะ Ghost (วิ่งทะลุกำแพงได้) เป็นเวลา 4 วินาที (ระวังอย่าให้หมดเวลาขณะอยู่ในกำแพง มิฉะนั้นจะตายทันที)
- **ศัตรู (ปีศาจ):**
  - **สีแดง (สายล่า):** จะเดินตามล่าผู้เล่นโดยใช้ระบบค้นหาเส้นทาง (A* Pathfinding Algorithm)
  - **สีส้ม (สายเฝ้าด่าน):** จะเดินสุ่มทิศทาง (Random Walker) เพื่อดักทางการหลบหนี
- **ปุ่มคำสั่ง:**
  - ใช้ **เมาส์คลิก** ที่ปุ่ม START หรือกด **SPACE / ENTER** เพื่อเริ่มเกม
  - กด **R** เมื่อจบด่าน (เพื่อไปด่านถัดไป) หรือเมื่อตาย (เพื่อเริ่มเล่นเลเวลเดิมใหม่)

## วิธีการติดตั้งและรัน
1. ติดตั้ง Python 3.11 ขึ้นไป (เนื่องจาก Python 3.14 อาจยังไม่รองรับ Pygame อย่างสมบูรณ์) : `winget install Python.Python.3.11`
2. สร้าง Virtual Environment (แนะนำ): `python -m venv venv`
3. Activate venv เพื่อเริ่มใช้งาน:
   - **สำหรับ Windows (Command Prompt):** `venv\Scripts\activate`
   - **สำหรับ Mac/Linux:** `source venv/bin/activate`
   > **⚠️ หมายเหตุสำหรับผู้ใช้ Windows PowerShell:** > หากรันคำสั่ง Activate แล้วติด Error สีแดง (ระบบบล็อกสคริปต์) ให้พิมพ์คำสั่งด้านล่างนี้เพื่อปลดล็อกสิทธิ์ก่อน:
   > `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser` 
   > (หากมีข้อความยืนยันให้พิมพ์ `Y` แล้วกด Enter จากนั้นลองคำสั่ง Activate อีกครั้ง)
4. ติดตั้ง dependencies: `pip install -r requirements.txt`
5. รันเกม: `python main.py`

## โครงสร้างและการประยุกต์ใช้ OOP / SOLID Principles
โปรเจกต์นี้พัฒนาด้วยภาษา Python โดยใช้ Pygame Framework และยึดหลักการเขียนโปรแกรมเชิงวัตถุ (OOP) ดังนี้:
- **Inheritance (การสืบทอด):** มีการสร้าง Base Class `Entity` เพื่อกำหนดคุณสมบัติพื้นฐาน (เช่น พิกัด x, y) และให้คลาสลูกอย่าง `Player` และ `Enemy` สืบทอดไปใช้งาน
- **Encapsulation (การห่อหุ้ม):** ซ่อนตัวแปรภายในคลาส (Protected attributes) และเข้าถึงข้อมูลผ่าน Getter / Setter (ใช้ `@property`) 
- **Polymorphism (การพ้องรูป):** คลาส `Player` และ `Enemy` มีการสืบทอดและเขียนทับ (Override) ฟังก์ชัน `update()` และ `draw()` จากคลาสแม่ เพื่อให้แต่ละคลาสมีพฤติกรรมการอัปเดตและการวาดกราฟิกที่แตกต่างกันตามหน้าที่ของตนเอง
- **Single Responsibility Principle (SRP):** มีการแบ่งแยกหน้าที่ของคลาสอย่างชัดเจน เช่น `MazeGenerator` ทำหน้าที่แค่สร้างและจัดการตรรกะเขาวงกต ในขณะที่ `Game` ทำหน้าที่รวบรวมและจัดการ Flow หลักของแอปพลิเคชัน
